Jaeger OTLP Auto-Fix Installation

This tool automatically fixes the issue with negative values in unsigned integer fields in OTLP traces that cause errors when uploading to Jaeger.

The Issue

When uploading OTLP traces to Jaeger, you might encounter this error:

Error converting traces to OTLP (�)

This happens because:

  1. The OpenTelemetry Java SDK sometimes generates traces with negative values for fields like droppedAttributesCount
  2. According to the OTLP specification, these fields should be unsigned integers (uint32)
  3. Jaeger's parser rejects these values with a cryptic error message

Installation Options

Option 1: Browser Extension (Recommended)

  1. Install the Tampermonkey extension for your browser
  2. Click the button below to install the auto-fix script:

    Install Auto-Fix Script

  3. Tampermonkey will open a new tab showing the script. Click "Install" to confirm
  4. The fix is now active whenever you use Jaeger UI

Option 2: Manual Script Injection

  1. Open your Jaeger UI in the browser
  2. Press F12 to open developer tools
  3. Go to the Console tab
  4. Copy and paste the contents of auto-fix-otlp.js into the console
  5. Press Enter to run the script
  6. The fix will be active for the current session

Fields Fixed

The script automatically fixes negative values in the following fields:

  • droppedAttributesCount
  • droppedEventsCount
  • droppedLinksCount
  • droppedLabelsCount
